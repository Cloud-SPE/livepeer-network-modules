# Manifest example

Generated by `go run ./examples/minimal-e2e/...` and rewritten here verbatim. Field values (eth_address, signature) change per run — the *shape* is what's stable.

```json
{
  "schema_version": "3.0.1",
  "eth_address": "0x3ecb9b37a1fded7ce5e21dd90ca8a81879491016",
  "issued_at": "2026-04-29T00:53:20Z",
  "nodes": [
    {
      "id": "ai-east",
      "url": "https://ai-east.example.com:8935",
      "extra": { "geo": { "region": "us-east-1" } },
      "capabilities": [
        {
          "name": "openai:/v1/chat/completions",
          "work_unit": "token",
          "offerings": [
            { "id": "gpt-oss-20b", "price_per_work_unit_wei": "1000", "constraints": { "gpu": "l40s" } }
          ]
        },
        {
          "name": "openai:/v1/embeddings",
          "work_unit": "token",
          "offerings": [
            { "id": "text-embedding-3-small" }
          ]
        }
      ]
    },
    {
      "id": "ai-west",
      "url": "https://ai-west.example.com:8935",
      "capabilities": [
        {
          "name": "openai:/v1/chat/completions",
          "work_unit": "token",
          "offerings": [
            { "id": "gpt-oss-20b", "price_per_work_unit_wei": "1100" }
          ]
        }
      ]
    },
    {
      "id": "transcoder-1",
      "url": "https://orch.example.com:8935",
      "capabilities": [
        { "name": "livepeer:transcoder/h264", "work_unit": "frame" }
      ]
    }
  ],
  "signature": {
    "alg": "eth-personal-sign",
    "value": "0x40992373db2c<...130-hex...>1c",
    "signed_canonical_bytes_sha256": "0x<...64-hex...>"
  }
}
```

Notes:

- `schema_version` is emitted as `"3.0.1"` and validators accept the caret-compatible range `^3.0.1`.
- `eth_address` is lower-cased canonical form.
- `issued_at` is RFC3339 UTC, but it is not covered by the signature; the publisher stamps it at sign time.
- `offerings[]` is the priced-tier list under each capability. The `id` field is opaque to the registry — for AI workloads it's typically the model name (`gpt-oss-20b`); for video transcoding a preset; for streaming sessions a resolution/fps tier. Renamed from `models[]` in v3 (was `model[].id`).
- Node-level `extra` and offering-level `constraints` are opaque JSON objects. The registry validates only that they are objects with max nesting depth 10.
- `signature.alg` is exactly `"eth-personal-sign"`; the value recovers to the same `eth_address` over the canonical bytes (see [signature-scheme.md](../design-docs/signature-scheme.md)).
- The same manifest carries AI and transcoding capabilities side-by-side. The registry treats them identically — capability strings are opaque (see [workload-agnostic-strings.md](../design-docs/workload-agnostic-strings.md)).
